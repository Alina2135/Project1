# Інструкція для роботи з проєктом (CONTRIBUTING.md)

## Для користувачів (як запустити калькулятор)

1. Натисніть зелену кнопку **"Code"** → **"Download ZIP"**
2. Розархівуйте папку
3. Запустіть `Pr1_Biriuk.py`

## Для розробників (як запропонувати зміни)

1. Створіть власну копію репозиторію, натиснувши кнопку **Fork** у  верхній частині сторінки GitHub(копія репозиторію у ваш акаунт).
2. Склонуйте свою копію:
   ```bash
   git clone https://github.com/your-username/Project1.git
3. **Створіть нову гілку** 
 ```bash 
 git checkout -b feature/назва-функції 
 ``` 
 Використовуйте зрозумілі назви гілок: 
 `feature/`, `fix/`, `docs/`, `refactor/` тощо. 
## Внесіть зміни та зробіть коміт
 ```bash 
 git add . 
 git commit -m "feat: додано форму реєстрації" 
 ``` 
 Формат повідомлень комітів: 
 ``` 
 <тип>: короткий опис 
 ``` 
 Типи: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`. 
## Відправте зміни
 ```bash 
 git push origin feature/назва-функції 
 ``` 
## Створіть Pull Request
   
 На сторінці вашого форку натисніть **New Pull Request** і виберіть  гілку, яку хочете об’єднати.

## Правила стилю коду 
* Дотримуйтесь eslint-конфігурації, яка йде з проєктом. 
* Використовуйте **2 пробіли** для відступів. 
* Назви змінних – у форматі `camelCase`. 
* Компоненти React називайте з великої літери. 
* Не додавайте непотрібні залежності. 
Перед створенням PR перевірте стиль коду: 
```bash 
npm run lint 
```

## Тестування 
Якщо ви додали нову функцію, обов’язково додайте тести. 
Тестовий запуск: 
```bash 
npm test 
```

## Репорти про помилки та пропозиції 
Якщо ви знайшли помилку або хочете запропонувати покращення,  створіть **issue**: 
1. Натисніть «Issues» → «New issue». 
2. Опишіть проблему або пропозицію якомога детальніше. 
3. Додайте скріншоти або приклади коду, якщо це можливо.
